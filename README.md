# OmniRoute on Coolify

Deployment of the [`diegosouzapw/omniroute`](https://hub.docker.com/r/diegosouzapw/omniroute)
image (unified AI proxy — one endpoint for 237 LLM providers, source:
[github.com/diegosouzapw/OmniRoute](https://github.com/diegosouzapw/OmniRoute)).

Image facts (verified against the registry config, tag `latest` = 3.8.46):

| What | Value |
|---|---|
| Application port | **20128** (dashboard + API together) |
| Data directory | `/app/data` (SQLite, logs, backups) |
| Container user | `node` (uid 1000) — entrypoint checks permissions on `/app/data` |
| Healthcheck | built-in: `node healthcheck.mjs` every 30 s |
| Heap limit | 1024 MB (`OMNIROUTE_MEMORY_MB` + `NODE_OPTIONS`) |
| Architectures | amd64, arm64 |

## Deployment (recommended): Docker Compose in Coolify

1. Coolify → project → **New Resource → Docker Compose Empty**, paste the
   contents of `docker-compose.yml`.
2. Coolify auto-generates all secrets (`SERVICE_*` magic variables) and a
   domain pointing at port 20128. Change the domain in the `omniroute`
   service settings (it must be **HTTPS** — see `AUTH_COOKIE_SECURE` below).
3. **Deploy.** The container reports `healthy` after ~15–30 s.
4. First dashboard login: `https://<your-domain>` — find the password in
   Coolify → **Environment Variables** → the value of
   `SERVICE_PASSWORD_OMNIROUTE` (injected as `INITIAL_PASSWORD`).
   After logging in change it: **Dashboard → Settings → Security**.
5. Add providers (API keys, OAuth accounts) in **Dashboard → Providers**
   — they are stored encrypted in SQLite (`API_KEY_SECRET`), not in env.

Proxy endpoint for tools (Claude Code, Cursor, Cline, etc.):
`https://<your-domain>/v1` + a key from **Dashboard → API Keys**
(the compose sets `REQUIRE_API_KEY=true`, so a key is mandatory).

## Alternative: single "Docker Image" / Dockerfile resource

- **Docker Image**: use `diegosouzapw/omniroute:latest`, port 20128,
  add a Volume for `/app/data` and paste the variables from `.env.example`
  (sections 1–4 are the minimum; generate the secrets with `openssl` as
  described in the comments).
- **Dockerfile**: use the bundled `Dockerfile` (a thin wrapper around the
  image), the rest as above. Without compose there is no Redis — in that
  case **remove/comment out** `REDIS_URL` (pointing it at a non-existent
  Redis floods the logs with ioredis errors; the app falls back to the
  built-in in-memory limiter).

## Environment variables — cheat sheet

Required secrets (auto-generated by Coolify in the compose deployment):

| Variable | Role | Manual generation |
|---|---|---|
| `JWT_SECRET` | dashboard session signing | `openssl rand -base64 48` |
| `API_KEY_SECRET` | encryption of provider keys in the DB | `openssl rand -hex 32` |
| `STORAGE_ENCRYPTION_KEY` | SQLite at-rest encryption (optional, recommended) | `openssl rand -hex 32` |
| `INITIAL_PASSWORD` | first-login password (bootstrap only) | any strong value |

The keys go through scrypt (a KDF), so they don't have to be hex — any
strong random string works. **Note:** set `STORAGE_ENCRYPTION_KEY` once —
changing it after the first start cuts off access to the encrypted
database (rotate via `STORAGE_ENCRYPTION_KEY_VERSION`).

Key settings for running behind a reverse proxy (Coolify/Traefik):

| Variable | Value | Why |
|---|---|---|
| `AUTH_COOKIE_SECURE` | `true` | required behind HTTPS, otherwise login breaks; on plain HTTP it must be `false` |
| `NEXT_PUBLIC_BASE_URL` | public `https://...` | OAuth callbacks and UI links |
| `BASE_URL` | `http://localhost:20128` | internal sync jobs — intentionally loopback |
| `REQUIRE_API_KEY` | `true` | a public instance should not proxy anonymously |

Full list (2000+ documented lines):
[.env.example in the OmniRoute repo](https://github.com/diegosouzapw/OmniRoute/blob/main/.env.example).

## Image variants and updates

- `latest` / `3.8.46` — standard; `latest-web` / `3.8.46-web` — with
  Chromium/Playwright, required only for web-cookie providers
  (`gemini-web`, `claude-web`, `claude-turnstile`); ~2x bigger image.
- For production pin a specific tag and bump deliberately; the project
  releases very frequently.
- Updating in Coolify: **Redeploy** (with `latest` and pull enabled).
  Database migrations run automatically on startup; an automatic SQLite
  backup is taken beforehand (`DISABLE_SQLITE_AUTO_BACKUP=false`).

## Persistence and permissions

Data lives in the `omniroute-data` named volume → `/app/data`. The
container runs as `node` (uid 1000); with a named volume Docker handles
the permissions. If you switch to a host bind-mount, set the owner:
`chown -R 1000:1000 <directory>` — otherwise the entrypoint
(`check-permissions.sh`) stops the startup with fix instructions.
